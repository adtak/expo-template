# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A setup script (`setup.sh`) and `template/` directory for bootstrapping new Expo projects. It covers only what `create expo-app` and `eas build:configure` don't provide — formatting, testing, CI, and Claude Code workflow config.

## Repository Structure

- `setup.sh` — run from inside a new Expo project root; copies `template/` files and mutates `package.json` / `app.json`
- `template/` — files copied verbatim into the target project:
  - `.claude/` — CLAUDE.md, agents, and skills for the target project
  - `.github/` — GitHub Actions CI workflow and Dependabot config
  - `biome.json` — Biome formatter config (formatting + import sorting only; linting left to ESLint)
  - `jest.config.js` — Jest config using `jest-expo` preset with `@/` alias

## How `setup.sh` Works

1. Validates it's running inside an Expo project (`app.json` with `expo` key)
2. Copies `template/` files into the current directory
3. Installs devDependencies via `pnpm add -D`
4. Adds scripts to `package.json` (`go`, `format`, `format:check`, `test`)
5. Mutates `app.json`: adds `ios.bundleIdentifier`, `expo-build-properties` plugin, and `extra.eas.projectId` placeholder
6. Removes `app-example` line from `.gitignore`
7. Runs `pnpm format` to format all existing code

## Modifying the Template

When changing what gets copied into target projects, edit files under `template/`. When changing setup behavior (dependencies, `app.json` mutations, scripts), edit `setup.sh`.

`eas.json` is intentionally absent — it's generated by `eas build:configure` in the target project after setup.
